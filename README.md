# HRBP-AI-
 ●人力预算：每年在9月份根据业务需求、公司成本管控的要求等，制定下一年每个月在岗人，作为下一年的预算；实际人力执行时，基本要求是：①全年实际平均在岗不超过预算全年平均在岗（by月在岗相加除以12个月）；②12月实际在岗不超过预算12月在岗 ●人力预估：第二年实际进行过程中，BP需要定期根据实际的人力流入、流出节奏，刷新部门by月的人力预估，并记录已发生月份的实际在岗人力，动态监控部门by月人力的变化，调整流入、流出节奏，确保全年平均在岗和年末实际在岗满足预算规划的要求 HRBP 每月要从多个系统人工取数。同时各个系统的数据未能做到实时准确更新，所以BP仍需要花大量的时间精力 线下用Excel与各方对齐。 本产品把这件事搬上 Web：自动汇集数据 → 自动计算预估、出表→ 人只做确认和调节。

## 原型 · 空态版（index.html）

- **单文件离线原型**：直接用浏览器打开 `index.html` 即可（无需构建/后端）。
- **空态约定（高压线）**：所有数字须由真实 API / 数据源提供；数据源未接入时**单元格一律留空，绝不编造/补零**。运算引擎"识空"——缺源数据时合计/预估链/年均同样留空。
- **结构**：看板0（人力管理要求·导入PDF内嵌）/ 看板1（部门维度集成表·主看板）/ 看板2（预算看板·其中求和+⊕分支）/ 看板3（社招滚动台账+待流入-社招面板）；年份页签（2025/2026/2027，可"＋新增年份"生成空模板）；调节项±分支（备注必填留痕）；双层数据（口径快照/实时层）；审计日志；AI 助手（RAG 问答·走人走的门）；PRD 覆盖清单页。
- **待接入数据源**：KPI系统(zhaopin)/员工宽表(diy)/HR数仓/招聘系统/招聘需求系统；未批期间走文件上传兜底。

## 管理后台 · demo（admin.html）

- 直接用浏览器打开 `admin.html`。
- **账号与权限**：iOA 账号 × 角色（管理员/HRBP·可编辑/领导·只读）× 部门空间 × 看板0-3 访问勾选 × 启停；可添加账号（重复拦截）。
- **项目配置**：看板1 指标行按分区管理——项目名称行内可改；**可手动添加**（⊕分支开关）；**前期数据解绑**（开=定义变更不回溯已发生月、历史按原口径封存；关=联动刷新历史）；可新增项目（初始数据为空，待接源/待录入，不编造）。
- **配置审计日志**：所有配置变更留痕。
